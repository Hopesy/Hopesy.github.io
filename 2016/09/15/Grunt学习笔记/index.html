Grunt学习笔记

  • grunt


前言

Grunt可以轻松搭建自动化的WEB前端开发环境，Grunt和grunt插件都是基于nodejs来运行的,jQuery在使用grunt，bootstrap在使用grunt，百度UEditor在使用grunt，你没有理由不学、不用。此外，Gulp也是一个不错的选择。

起步

1.安装

npm install -g grunt //项目需要时安装
npm install -g grunt-cli //命令行使用grunt命令

2.配置
项目目录下npm install grunt
新建Gruntfile.js并进行配置(文件名称一致)

//包装函数
module.exports =function (grunt){
//任务配置，所有插件的配置信息
grunt.initConfig({
//获取package.json的信息
pkg: grunt.file.readJSON(‘package.json’)
});
//告诉grunt当我们在终端中输入grunt时需要做些什么(注意先后顺序)
grunt.registerTask(‘default’,[]);
};

3.使用
运行grunt命令

常用插件介绍(contrib前缀为grunt团队贡献的插件)

Grunt官网插件列表页面:http://www.gruntjs.net/plugins
Contrib-jshint——javascript语法错误检查；
Contrib-watch——实时监控文件变化、调用相应的任务重新执行；
Contrib-clean——清空文件、文件夹；
Contrib-uglify——压缩javascript代码
Contrib-copy——复制文件、文件夹
Contrib-concat——合并多个文件的代码到一个文件中
karma——前端自动化测试工具

使用uglify插件（压缩javascript代码**.min.js）

1.插件安装
npm install grunt-contrib-uglify –save-dev
2.在grunt.initConfig方法中配置 uglify 的配置参数


uglify: {
options: {
banner: ‘/! <%= pkg.name %> - v<%= pkg.version %> - ‘ +
‘<%= grunt.template.today(“yyyy-mm-dd”) %> /‘
},
my_target: {
files: {
‘build/output.min.js’: [‘src/test.js’]
}
}
}
});

3.加载插件(如果现在不使用就在registerTask写)

//告诉grunt我们将要使用插件
grunt.loadNpmTasks(‘grunt-contrib-uglify’);
//告诉grunt当我们在终端中输入grunt时需要做些什么(注意先后顺序)
grunt.registerTask(“default”,[‘uglify’]);

结语

Grunt插件的用法大同小异，具体用法查看官方文档
