Mongo Db常用命令

  • MongoDb





Mong Db常用命令

启动方式

1.默认启动(指定文件目录)


mongod –dbpath “d:\software\MongoDB\Server\3.0\data”
下次启动只需: mongo.exe

2.指定配置文件启动


mongod –config “d:\software\MongoDB\Server\3.0\conf\mongod.conf”‘

配置文件常用选项

可以是相对路径也可以是绝对路径


1.日志文件位置
logpath=/var/log/mongo/mongod.log
2.以追加方式写入日志
logappend=true
3.是否以守护进程方式运行(WINDOWS无效)
fork = true
4.默认27017
port = 27017
5.数据库文件位置
dbpath=/var/lib/mongo
6.启用定期记录CPU利用率和 I/O 等待
cpu = true
7.是否以安全认证方式运行，默认是不认证的非安全方式
noauth = true
auth = true
8.详细记录输出
verbose = true

常用命令

1.打开连接(默认端口27017无改动时:mongo)
mongo localhost:port
2.关闭连接
use admin
db.shutdownServer()
exit
3.查看数据库列表
show dbs
4.切换（选择、创建）数据库
use db_name
5.插入数据（JSON格式)
db.collection_name.insert({“key”:”value”})
默认情况下自动生成-id,也可以自己指定，但不能重复
db.collection_name.insert({“key”:”value”,_id:1})
6.查看当前数据库中的表格（首先选择数据库)
show collections ||show tables
7.查看表格中的数据(查询条件为空返回所有)
db.collection_name.find()
8.指定条件查询(条件为JSON)
db.collection_name.find({“x”:1})
9.多条插入（删除……)
for(i=2;i<20;i++) db.collection_name.insert({x:i})

10.计数
db.collection_name.find().count()
11.跳过：skip(count)限制：limit(count)排序：sort(JSON)
db.collection_name.find().skip(3).limit(5).sort({“x”:1})
从当前表的第四条数据开始，依次查询五条数据，并按照x的值升序排序
12.数据更新
db.collection_name.update({x:1},{x:99})
13.部分字段更新$set
db.collection_name.insert({x:1},{y:99},{z:100})
将z=100的数据中字段x改为99
db.collection_name.update({z:100},{x:99})
错误：y和z将会被覆盖掉，只剩下x字段
db.collection_name.update({z:100},{$set{x:99}})
正确：只更新x字段，其他字段不变
14.更新时自动创建不存在的数据（添加第三个参数True)
db.collection_name.update({x:10},{x:100},true)
15.批量更新（update的第四个参数为true)
db.collection_name.update({x:10},{$set{x:100}},false,true)
备注:为防止误操作，必须使用$set
16.数据的删除
db.collection_name.remove({x:1})
备注:为防止误操作，remove()必须传参
17.删除某个表
db.collection_name.drop()
18.给表添加索引
db.collection_name.ensureIndex({x:1})
备注:x为1表示正向排序，为0表示逆向排序
19.获取某张表的索引
db.collection_name.getIndexes()
20.索引
索引可以重复创建，对于已经创建的索引会返回成功
_id:默认自动创建的索引
多建索引：值具有多个记录，例如数组
单键索引:值为单一的值，例如字符串，日期
过期索引:一段时间后索引消失，并且数据也会删除
expireAfterSeconds:1010 存储在过期索引字段的值必须是指定的时间类型，即必须是ISODate或ISODate数组
不能使用时间戳，否则不能自动删除 如果使用了ISODate数组，按照最小的时间进行删除
过期索引不能是复合索引 删除程序的时间间隔为60秒
21.索引
建立索引:db.collection_name.ensureIndex({x:1})
建立索引(指定名字):db.collection_name.ensureIndex({x:1},{name:”apple”})
删除索引:db.collection_name.dropIndex({x:1})
删除索引(使用名字):db.collection_name.dropIndex({name:”apple”})
唯一性,unique指定(指定后不能插入两条具有索引的相同的数据)
db.collection_name.ensureIndex({x:1},{unique:true})
稀疏性（默认不稀疏，稀疏性的不同代表了处理索引中存在字段中不存在的数据的两种
方法，指定为true时如果字段内不包括索引字段不会创建索引)
db.collection_name.ensureIndex({x:1},{parse:true})
$exist字段
db.collection_name.find({x:{$exist:true}})
21.全文索引
每个表中只允许创建一个全文索引
建立
db.collection_name.ensureIndex({key:”text”})
db.collection_name.ensureIndex({key1:”text”,key2:”text”})
db.collection_name.ensureIndex({“$**”:”text”}) 查询
db.collection_name.find({$text:{$search:”coffee”}})
多个关键词全文查询（或方式，只要包含一个关键词就行)
db.collection_name.find({$text:{$search:”coffee banana”}}) 过滤查询（-表示不包括某个关键词）
db.collection_name.find({$text:{$search:”coffee -apple”}})
多个关键词全文查询（与方式)
db.collection_name.find({$text:{$search:”\”coffee\” \”banana\””}}) 相似度查询
$mata操作符:{score:{$meta:”textScore”}}写在查询条件后面可以返回返回结果的
相似度与sort一起使用，可以达到很好的实用效果
db.collection_name.find({$text:{$search:”\”coffee\” \”banana\””}}，{score:{$meta:”textScore”}})
.sort({score:{$meta:”textScore”}})
使用限制
每次查询，只能制定一个$text查询
$text查询不能出现在$nor中
查询中如果包含了$text,hint将不起作用
22.地理位置索引
2d索引，用于存储和查找平面上的点
2d sphere 用于存储和查找球面上的点
查找方式:查找距离某个点一定距离的点；查找包含在某区域内的点
23.创建2d索引
db.collection_name.ensureIndex({w:”2d”})
db.collection_name.insert({w:[100,100]})
备注：使用位置表示方式：经纬度[经度，纬度]
取值范围：经度[-180,180]纬度[-90,90]
24.2D索引查询(near)
db.collection_name.find({w:{$near:[1,1]}})
db.collection_name.find({w:{$near:[1,1],$maxDistance:10}})
备注:near查询返回100个距离最近的点,可以使用maxDistance限制最大距离
25.2D索引查询(geoWithin)
db.collection_name.find({w:{geoWithin:{$box:[[0,0],[3,3]]}}})
$box 矩形；{$box:[[,],[,]]}矩形左边界和右边界
$center 圆形；{$center:[[,],r]} 圆心和半径
$polygon 多边形 {$polygon:[[,],[,],[,]]}多边形的坐标点
26.2D索引查询(goNear)
goNear使用runCommand进行查询
db.runCommand({geoNear:”collection_name”,near:[1,2],maxDistance:10,num:1})
备注:db.runCommand({geoNear:]}
查询方式与2d索引查询方式差不多
28.mongostat工具
mongostat -h 127.0.0.1:27017
29.profil集合
profil集合石mongoDb的慢操作日志，设置级别后会记录操作
db.getProfilingLevel()
db.getProfilingStatus()
级别(was)为0时不会记录任何操作，级别为1时记录超过slowms的操作，级别为2是都记录
db.getProfilingLevel(s2)
30.explain
三种模式queryPlanner、executionStats、allPlansExecution
db.collection_name.find({x:1}).explain()
31.用户验证 创建语法:createUser
*{user:”“ ,
pwd: ““,
customData:{<anyinformation},
roles:[{role:”“,db: ““}]}
角色类型:内建类型（read,readWrite,dbAdmin,dbOwner,userAdmin)
